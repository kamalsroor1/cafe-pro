# 🗄️ Database Schema — Cafe Pro ERP

> All tables use `id` (BIGINT UNSIGNED AUTO INCREMENT) as primary key and `timestamps` (`created_at`, `updated_at`) unless noted.  
> Soft-delete tables include `deleted_at`.

## 7. `categories`

```sql
id              BIGINT UNSIGNED PK
name            VARCHAR(255)
slug            VARCHAR(255) UNIQUE
parent_id       BIGINT UNSIGNED NULL FK → categories.id
description     TEXT NULL
image           VARCHAR(255) NULL
sort_order      INT DEFAULT 0
is_active       BOOLEAN DEFAULT TRUE
created_at      TIMESTAMP
updated_at      TIMESTAMP
```

> Self-referential: `parent_id` = NULL means top-level category.

## 11. `product_ingredient` *(Recipe / BOM Table)*

```sql
id              BIGINT UNSIGNED PK
product_id      BIGINT UNSIGNED FK → products.id
ingredient_id   BIGINT UNSIGNED FK → ingredients.id
amount_needed   DECIMAL(10,3)            -- Amount consumed per 1 unit of product
created_at      TIMESTAMP
updated_at      TIMESTAMP

UNIQUE KEY (product_id, ingredient_id)
```

> **Example:** 1 Latte = 20g Coffee Beans + 200ml Milk  
> Row 1: product_id=5, ingredient_id=1 (beans), amount_needed=20  
> Row 2: product_id=5, ingredient_id=2 (milk), amount_needed=200

## 12. `shifts`

```sql
id                  BIGINT UNSIGNED PK
user_id             BIGINT UNSIGNED FK → users.id   -- The cashier
opening_balance     DECIMAL(10,2)                   -- Cash in drawer at start
closing_balance     DECIMAL(10,2) NULL              -- Actual cash at end
expected_balance    DECIMAL(10,2) NULL              -- System-calculated expected
difference          DECIMAL(10,2) NULL              -- closing - expected (Over/Short)
status              ENUM('open','closed') DEFAULT 'open'
notes               TEXT NULL
opened_at           TIMESTAMP
closed_at           TIMESTAMP NULL
created_at          TIMESTAMP
updated_at          TIMESTAMP
```

## Key Indexes (Performance)

```sql
-- Orders: frequently filtered by shift and status
INDEX idx_orders_shift_id (shift_id)
INDEX idx_orders_status (status)
INDEX idx_orders_created_at (created_at)

-- Order items: joined on every order fetch
INDEX idx_order_items_order_id (order_id)

-- Ingredients: stock checks
INDEX idx_ingredients_stock_qty (stock_qty)

-- Expenses: date range queries
INDEX idx_expenses_date (date)
INDEX idx_expenses_shift_id (shift_id)
```
